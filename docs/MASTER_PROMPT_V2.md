AntiRibet BUKAN hanya:
- QR menu restoran,
- aplikasi kasir biasa,
- website builder biasa,
- sistem booking saja,
- sistem antrean saja,
- aplikasi restoran saja.

AntiRibet adalah platform all-in-one untuk:
- UMKM,
- F&B,
- retail,
- bisnis jasa,
- booking-based business,
- service-based business,
- rental business,
- professional service business,
- dan bisnis custom lainnya.

AntiRibet menyediakan:
- Company profile platform
- Website utama AntiRibet
- Register merchant
- Merchant onboarding wizard
- Mini website untuk setiap merchant
- Katalog produk atau layanan
- Kasir / POS universal
- QR order

- Customer database / CRM sederhana
- Resource management

- Top up saldo merchant

- Refund fee
- Audit log

- Merchant health score

- Full-stack Flutter + Laravel API architecture
- PostgreSQL database architecture

- Roadmap produk dan teknis

Model bisnis utama:
Merchant top up saldo.
Saldo dipotong Rp500 untuk setiap transaksi berhasil.
Tidak ada biaya bulanan wajib.
Tidak ada premium domain.
Tidak ada premium subdomain.
Tidak ada biaya per produk.
Tidak ada biaya per item.
Fee Rp500 adalah per transaksi berhasil.

Untuk MVP:
Customer membayar langsung ke merchant.
AntiRibet tidak memegang uang penjualan customer.
Merchant membayar AntiRibet melalui saldo deposit.
AntiRibet mendapatkan revenue saat saldo merchant dipotong Rp500 per transaksi berhasil.

Tech stack final:
Frontend: Flutter untuk Web, Android, iOS.
Backend: Laravel API.
Database: PostgreSQL.
Queue/cache: Redis.
Realtime: Laravel Reverb / WebSocket.
Auth: Laravel Sanctum API Token.
Storage: Local storage untuk MVP, S3-compatible storage untuk production.
State management Flutter: Riverpod.
Routing Flutter: go_router.
HTTP client Flutter: Dio.
Model serialization: freezed + json_serializable.
Mobile storage: flutter_secure_storage + shared_preferences.
QR: qr_flutter + mobile_scanner.
Charts: fl_chart.
PDF/Print: pdf + printing.

4. REGISTER MERCHANT FLOW
============================================================

Buat alur register merchant dengan sangat detail.

Register merchant adalah alur merchant membuat akun dan mendaftarkan bisnis pertama.

4.1 Entry Point
Merchant bisa masuk register dari:
- tombol CTA di landing page,
- halaman pricing,
- halaman demo,
- referral link,
- campaign link,
- direct URL /register-merchant.

4.2 Register Form Step 1 — Owner Account
Field:
- nama owner
- email
- nomor WhatsApp
- password
- konfirmasi password
- persetujuan terms & privacy

Validasi:
- email wajib dan unik,
- nomor WhatsApp wajib,
- password minimal aman,
- terms wajib dicentang.

Alur:
Merchant isi form owner
↓
Flutter kirim request ke Laravel API
↓
Laravel validasi data
↓
Laravel membuat user owner
↓
Laravel mengirim OTP/email verification jika diperlukan
↓
Jika sukses, lanjut ke step business info

4.3 Register Form Step 2 — Business Basic Info
Field:
- nama bisnis
- jenis bisnis
- kategori bisnis
- alamat bisnis
- kota
- nomor WhatsApp bisnis
- slug/link bisnis
- logo opsional

kopi-senja → antiribet.id/kopi-senja

Validasi:
- nama bisnis wajib,
- jenis bisnis wajib,
- slug unik,
- slug hanya huruf kecil, angka, dash,
- nomor bisnis valid.

Alur:
Merchant isi data bisnis
↓
Sistem mengecek slug tersedia

Laravel membuat business

Laravel membuat branch default “Cabang Utama”

============================================================

Tujuan:

Step onboarding:

5.1 Step 1 — Lengkapi Profil Bisnis

- banner
- deskripsi
- jam buka

- WhatsApp
- Instagram opsional
- metode pembayaran manual

Output:

Mini website punya data dasar.

5.2 Step 2 — Tambah Catalog Pertama
Merchant diminta menambah minimal 3-5 item:
- nama item
- tipe item
- kategori
- harga
- foto opsional
- status tersedia

Jika merchant punya file Excel/CSV, sediakan import.

Output:
Catalog siap dipakai POS dan mini website.

5.3 Step 3 — Pilih Modul Aktif
Merchant memilih modul:
- Kasir/POS
- QR Order
- Booking
- Antrean
- Invoice
- Customer database
- Resource

Default untuk mode sederhana:
- mini website
- catalog
- POS
- transaction
- report
- wallet

5.4 Step 4 — Setup Resource Jika Diperlukan
Jika merchant memilih QR order:
buat meja/resource.

Jika merchant memilih booking:
buat staff/resource.

Jika merchant memilih rental:
buat item/resource.

Output:
Resource siap untuk QR, booking, queue, atau rental.

5.5 Step 5 — Simulasi Transaksi
Merchant diajak mencoba transaksi dummy:
- pilih item,
- checkout,
- lihat struk,
- lihat laporan,
- lihat saldo tidak dipotong jika mode demo.

Tujuan:
merchant paham sistem sebelum transaksi real.

5.6 Step 6 — Trial Activated
Merchant mendapat:
- trial quota 50 transaksi pertama,
- status trial_active,
- reminder bahwa setelah trial habis perlu top up.

5.7 Step 7 — Website Aktif
Mini website merchant aktif:
antiribet.id/{businessSlug}

Merchant bisa:
- copy link,
- share ke WhatsApp,
- download QR,
- mulai menerima transaksi.
