## 2.1 Tech Stack Final Versi Flutter

2.1.1 Frontend utama: Flutter (Dart)
2.1.3 Platform: Web, Android, iOS
2.1.4 Backend: Laravel API
2.1.5 Database: PostgreSQL

## 2.15 Multi-Tenant Logic

2.15.1 Satu database untuk semua merchant.
2.15.2 Semua data punya `business_id`.

---

## 2.16 Auth Flow

Flutter login -> Laravel validasi -> Buat token Sanctum -> Flutter simpan token (Secure Storage) -> Kirim token Bearer via Dio -> Laravel validasi.

---

## 2.17 Authorization (RBAC)

Role, Permission, Policy, Gate, Super admin access, Business owner access, Manager access, Cashier access, Staff access.

---

## 2.18 Wallet Technical Logic

Merchant wallet menyimpan Balance.
Fee Rp500 per transaksi berhasil dideduksi via WalletTransaction.
*Penting:* Menggunakan Wallet Lock (`lockForUpdate()`) agar tidak terjadi Double Charge saat diakses bersamaan.

---

## 2.19 POS Technical Flow

Flutter POS cart -> Checkout request -> Laravel validate item -> Recalculate total -> Create transaction & payment -> Charge platform fee Rp500 -> Return receipt.

---

## 2.20 QR Order Technical Flow

Customer scan QR -> Flutter Web -> Load catalog -> Checkout -> Order pending -> Diterima Kasir -> Transaksi Paid -> Fee Rp500 terpotong.

---

## 2.21 Booking & Queue Technical Flow

(Booking) Select resource & time -> Create booking -> DP.
(Queue) Ambil nomor -> Generate unique number -> Status Waiting -> Called -> Completed.

---
